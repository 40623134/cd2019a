  2. 每組將提供組長本網站倉儲協同管理權, 專門處理各組員在各分支或 master 分支中的 pull requests, 各組員必須透過以 ag1, ag2 .... 命名之分支對本倉儲改版, 確定內容無誤後, 再設法由組長處理各 pull requests 之內容合併, Github Pages 將設為與 master 分支對應, 各組員之評分將依據各 pull requests 中所列出之學號與內容判定.
  3. 請各組員勿將影片檔、圖片檔案或各式檔案送至本倉儲, 影片檔案請上傳至 Youtube, 圖片檔案請送至各組倉儲網站, 各式檔案則請送至各組員 Google Drive 區, 並以嵌入 (iframe), 將影片放入本網站, 其餘圖片或檔案則透過 img 或 a 標註放入各相關頁面.

專案結論

設計與繪圖

零組件尺寸分析

1.球桌

我們是參考火球國際公開賽桌球場的規格做基準，基本上對尺寸做了一點微調。

球桌剛成形.jpg

增加為了讓回球機構能運作的平面，且球門前的球員行程並沒有想像中的寬，故在角落處增加圓弧面。

球桌完成.jpg

2.球員

外型依照老師提供的造型作為使用，我們將孔洞放大便於組合與安裝。

球員.jpg

3.球桿

參考火球國際公開賽桌將球桿分為一、二、三、五人，桿上突起部位為安裝球員處。

單人、雙人桿在末端都向外延伸了一個10mm的圓柱，用來限制球員的行程和防止球桿從孔洞掉落。

單人桿.jpg 、 雙人桿.jpg

三人、五人桿因為安裝球員處的圓柱能達到限制形成的效果，所以並沒有向外延伸圓柱。

三人桿,jpg 、 五人桿.jpg

4.圓盤

這是剛開始使用的回球機構的圓盤。

舊圓盤,jpg 、 舊圓盤剖面.jpg

因為球會卡在球道末端而修改了錐形孔的圓心位置和圓盤大小及厚度。

新圓盤.jpg 、新圓盤剖面.jpg

5.握把

依照火球國際公開賽桌的握把去抓尺寸而繪製的樣子，但我們並沒有繪製銷孔和銷，只是單純利用塑膠可壓縮性而配合上去的。

握把.jpg

6.球道

剛開始設計出來的球道模型。

舊球道.jpg

球道的入口處因球道入口處太小，有機會進不了入口或彈出，因為入口處地板是平面所以球有機會停在此處。

舊球道入口.jpg

出口處平面部分過長，球有機會停置於此處。

舊球道出口.jpg

修改後的整體圖

新球道.jpg

修改後的入口處地板採用斜面、且入口增大，解決了入口卡球的問題。

新球道入口.jpg

出口修改為斜面到底，且增加圓弧面更容易讓球滾入圓盤。

新球道出口.jpg

7.套筒

為了讓球桿方便安裝，原本的孔洞與球桿設定為直徑15mm，但桿上的球員安裝處為直徑20mm，故將球桌的孔洞從15mm改為20mm，使球桿穿過孔洞之後安裝套筒就能使球桿固定於孔上也不會有安裝不便的問題。

套筒.jpg
